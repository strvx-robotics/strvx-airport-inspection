## Goal & acceptance (the PRD ¬ß14 run, as the definition of done)

Slice A is done when this exact run passes against **real SQLite persistence with CV stubbed**:

1. Admin creates an airport, 3 runways, zones, and a schedule (admin setup surface ‚Üí `inspection_store`).
2. The minimal scheduler **materializes** a 6 AM `Inspection` + one `InspectionJob` per runway ‚Äî **records only, never launches the drone** (startup catch-up tick or run-now).
3. An image is uploaded with a **manual zone tag** (+ EXIF-GPS when present); it ties to runway/zone; `geom_confidence ‚àà {gps, manual}` (never `pose` in Slice A).
4. `StubAirportDetector` produces `IssueCandidate` rows; they render with **confidence bands** (Low hidden by default, enforced at the API).
5. Inspector **approves / edits category (PRD ¬ß9.5)** ‚Üí creates a `Ticket` from the inspector-finalized text, seeded from **LLM-drafted text** (`draft_ticket()` stub).
6. **Reject requires a `RejectionReason`**, persisted to the immutable `issue_status_history` and denormalized on the candidate.
7. The issue screen shows a **git-style diff view** of immutable `ai_draft_text` vs the inspector-edited text (live before a ticket exists; vs `Ticket.description` after).
8. Maintenance marks **repaired**; inspector **closes**.
9. **Report exports** (HTML + JSON).
10. The **admin-only feedback-export** (`GET /inspection/feedback-export`) returns JSONL learning records (rejections + draft pairs).

**Global invariants** (must hold throughout): RBAC fail-closed; **no auto-ticket** ‚Äî the guarantee is structural and twofold: (a) `process_image` never calls any `*ticket*` store method (T24), and (b) a `Ticket` is only ever created by an authenticated human `create_ticket` command (T17/T30). The ISR guardrail rule `source='reasoner' ‚Üí Ok(auto=False)` is **left untouched**; it governs `AutonomyCommand`s and `draft_ticket()` returns *text* that never traverses it, so it is not the mechanism here ‚Äî keeping the existing guardrail test green (T24) is the framing **[FIX C2-missing]**. `inspection_store` **raises on write failure** (not the fail-open mission `SqliteStore` pattern); schema-first contracts pass `make check-contracts`; per-dir `docs.md` updated in the **same commit** as code; tenancy is forward-compat only ‚Äî `org_id` is a **nullable column on `airports`** (the tenancy root) + a typed `User.orgId`, with no tenancy logic anywhere.

- **PR-A is the universal unblocker** ‚Äî land it first; every other PR imports generated types and aligns string literals to its enums. A new enum not registered in `ENUM_CONST` (T3) KeyErrors the whole codegen ‚Äî T3 is the only `gen_contracts.py` edit.

- **The airport surface is gated three ways on `airport_enabled`** (`VALANOR_AIRPORT`): the store build (T8 ‚Üí `hub.inspection`), the scheduler lifespan (T27), and the single `include_router` (T29). - **`server.py` is edited by PR-B (gated hub store), PR-F (gated scheduler lifespan), PR-G (gated `include_router`)** ‚Äî additive, ~5-15 lines each. Run-now and report routes live on the gated router (PR-G/PR-H), not `server.py`, so the whole `/inspection/*` surface shares one gate.

- **`VIEW_INSPECTION` is the inspection/ticket read perm** (inspector+maintenance+admin), distinct from ISR `WATCH` ‚Äî maintenance can open its tickets (PRD ¬ß6) without leaking the ISR feed (design ¬ß7); gating reads on `WATCH` would have 403'd maintenance **[FIX C2-B1]**.
